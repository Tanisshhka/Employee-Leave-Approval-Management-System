Chain of Command Extension Simulation

D365 Pattern

What is Chain of Command?

Chain of Command is a D365 F&O pattern that allows developers to extend or override base class methods without modifying the original code. It ensures that the base logic still executes unless explicitly skipped.

Base Class: LeaveCalculation
class LeaveCalculation {
    public int calculateLeaveDays(
        Date startDate, Date endDate
    ) {
        int days = endDate - startDate + 1;
        return days;
    }
}
➔
Extended Class: LeaveCalculationExtension
[ExtensionOf(classStr(LeaveCalculation))]
final class LeaveCalculationExtension {
    public int calculateLeaveDays(
        Date startDate, Date endDate
    ) {
        int days = next calculateLeaveDays(
            startDate, endDate
        );
        days = days + this.addBonusDays();
        return days;
    }
    private int addBonusDays() {
        return 2; // 2 bonus days
    }
}

Event Handler Simulation

D365 Pattern

What are Event Handlers?

Event Handlers in D365 F&O are methods that execute in response to specific events like record insertion, update, deletion, or field changes. They enable validation, data manipulation, and side-effect logic without modifying the base table or form code.

OnChanged Event

Triggers when a field value changes. Used for real-time validation.

public void employeeId_OnChanged()
OnValidating Event

Triggers before saving to validate data integrity.

public void leaveRequest_OnValidating()
OnPost Event

Triggers after a database operation completes.

public void leaveRequest_OnInserted()

Workflow Approval Process

D365 Concept: Workflow Framework

Web Equivalent: JavaScript state management

Process Flow:

  • Employee submits request (Pending status)
  • Manager reviews in approval queue
  • Approve or Reject with comments
  • Status updated and logged
  • Notification sent to employee
// D365 Workflow Configuration
class LeaveRequestWorkflow
    extends WorkflowDocument
{
    public void submit()
    {
        this.status = LeaveRequestStatus::Pending;
        this.insert();
    }
    public void approve(UserId approver)
    {
        this.status = LeaveRequestStatus::Approved;
        this.approvedBy = approver;
        this.update();
    }
}

Chain of Command Extension

D365 Concept: Chain of Command Pattern

Web Equivalent: Class extension / Decorator

How it Works:

  • Base class defines core logic
  • Extension adds bonus features
  • next keyword calls base method
  • Original code stays unmodified
  • Multiple extensions can chain
// D365 Chain of Command
[ExtensionOf(classStr(LeaveCalculation))]
final class LeaveCalcExtension
{
    public int calculateLeaveDays(
        Date startDate, Date endDate
    ) {
        // Calls base, then extends
        int days = next calculateLeaveDays(
            startDate, endDate
        );
        return days + 2; // Bonus
    }
}
